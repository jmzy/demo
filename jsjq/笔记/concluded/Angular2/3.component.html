Anuglar2 组件

什么是 Anuglar2 组件 
组件是对数据和方法的简单封装
根组件 
根组件在src文件夹下app文件夹下
一般情况下、包含ts(逻辑)、html(视图)、css(样式) 组成
创建新组件 
创建命令 ng generate component component-name
src文件夹下app文件夹里的文件夹就是创建新的组件
初识组件当中内容 
src文件夹下的app.component.ts
@Component({
  selector: 'app-root',//接口
  templateUrl: './app.component.html',
  //template为文件下的html内容、templateUrl为连接
  styleUrls: ['./app.component.css']//同上为css的文件
})

export class AppComponent {//json类集
  title = 'app';//绑定数据:自定义对象和值
}

src文件夹下的app.component.html下使用数据就可以了
例：<h1>{{title}}</h1>
更改内容需要启动服务在根localhost：4200查看更改的内容
数据绑定 
模版绑定：
src文件夹下的app.component.html下
绑定需要对象的新组件、
若组建名为home、绑定的标签则为 <app-home></app-home>
绑定的新组件全部内容都会导入

属性绑定：
在模版绑定的基础下 HTML 使用了数据后
例：
模版绑定：<input value={{title}} />
属性绑定：<input [value]="title" />
模版班定和属性绑定的区别 
模版绑定只能使用字符串、属性绑定字符串和表达式
模版绑定使用得更多
双向绑定 two way binding

## Build
Run `ng build` to build the project. The build artifacts will 
be stored in the `dist/` directory. Use the `-prod` flag for a 
production build.

## Running unit tests
Run `ng test` to execute the unit tests via [Karma]
(https://karma-runner.github.io).

## Running end-to-end tests
Run `ng e2e` to execute the end-to-end tests via [Protractor]
(http://www.protractortest.org/).
Before running the tests make sure you are serving the app via 
`ng serve`.

## Further help

To get more help on the Angular CLI use `ng help` or go check 
out the [Angular CLI README]
(https://github.com/angular/angular-cli/blob/master/README.md)
