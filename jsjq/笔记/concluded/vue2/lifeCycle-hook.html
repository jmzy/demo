VUE2 生命周期钩子

下图展示了实例的生命周期。你不需要立马弄明白所有的东西， 不过随着你的不断学习和使用，它的参考价值会越来越高。 
new Vue({}) //实例化(传配置参数)
beforeCreate() //生命周期钩子
 -- Observe Data //数据的检测 和 监听的配置(get和set的过程)
 -- init Events  //初始化事件(写在HTML绑定的事件监听mothods)
create()
 -- Has 'el' option? 检测是否有元素挂载点
 -- (有,继续往下) Has 'template' option?
 -- (没有,就通过函数$mount()触发) when vm.$mount(el) is called (如果已经触发,再回到 有template选项)
-yes Compile template into render function -有,通过render方法去编译
-no Compile outerHtml as template -没有,通过outerHtml作为模版
beforeMount()
 -- Create vm.$el and replace 'el' with it 有没有el选项,对应的dom元素是不是能加载,能获取
mounted()
 --对数据的检测 when data changes 判断是否更新
beforeUpdate() 更新之前
 --Virtual DOM re-render and patch 涉及数据响应的机制,视栈队列(数据更新到DOM的时间差) 涉及到虚拟DOM(最早是react)
updata() 更新之后
 --when vm.$destroy() is called 实例销毁
beforeDestroy() 销毁之前
 -- Teardown watchers,child components and event listeners 会解除所有(事件绑定,数据监听,watch,销毁主键)
destroyed()
beforeCreate() 
场景:使用客户端判断是否初次登录,欢迎页或引导收藏等
create() 
数据加载成功,可以修改数据,但渲染没成功(主要是dom)
场景:欢迎页,客户数据加载,初始化,录播图实现,
判断登录后加载指定信息
判断登录后返回优惠页面(初次数据加载成功后判断是否修改数据)等
beforeMount() 
放弃了解,个人感觉使用场景有限
mounted() 
没有绑定可以手动绑定$mount('#app')
beforeUpdate() 
数据跟新前进行操作
updata() 
(不建议)可以在异步获取数据后进行操作
对数据更新做全局处理,具体属性更新可以用watch
beforeDestroy() 和 destroyed() 
组件销毁前和后使用
activated() 和 deactivated() 
组件<keep-alive>激活和移除时调用
errorCaptured()
watch() 
逻辑简单的可以直接使用computed()

JavaScript一种直译式脚本语言，是一种动态类型、弱类型、基于原型的语言，内置支持类型。它的解释器被称为JavaScript引擎，为浏览器的一部分，广泛用于客户端的脚本语言，最早是在HTML（标准通用标记语言下的一个应用）网页上使用，用来给HTML网页增加动态功能。 为了取得技术优势，微软推出了JScript，CEnvi推出ScriptEase，与JavaScript同样可在浏览器上运行。为了统一规格，因为JavaScript兼容于ECMA标准，因此也称为ECMAScript。 在1995年时，由Netscape公司的Brendan Eich，在网景导航者浏览器上首次设计实现而成。因为Netscape与Sun合作，Netscape管理层希望它外观看起来像Java，因此取名为JavaScript。但实际上它的语法风格与Self及Scheme较为接近。
