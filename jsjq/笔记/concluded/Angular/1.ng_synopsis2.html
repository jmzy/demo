Controller使用过程中的注意点 
1.不要复用Controller，一个控制器正常只负责一小块视图
  允许存在子Controller、子Controller是可以继承父Controller的一些命令
2.不要在Controller中操作DOM，这不是控制器的职责
  操作DOM速度是非常低、会让浏览器重绘、重新布局
3.不要在Controller里面做数据格式化，ng有很好用的表单控件
4.不要在Controller里面做数据过滤操作，ng有$filter服务
5.一般来说，Controller是不会互相调用的，控制器之间的交互会通过事件进行
神奇的$scope 
AngularJS 的 MVC 是借助于$scope实现的
全部借助于作用域实现的

神奇的$scope
1.是一个POJO（Plain Old JavaScript Object）
2.有一些工具方法 $swatch()/$apply()
3.是表达式的执行环境（或者叫作用域）
4.是一个树形结构、与 DOM 标签平行
5.子 $scope 对象会继承父 $scope 上的属性和方法
6.每个 Angular 应用只有一个根 $scope 对象(一般位于ng-app上)
7.$scope 可以传播事件、类似 DOM 事件 、可以向上也可以向下
8.$scope 不仅是 MVC 的基础、也是实现双向数据绑定的基础
9.可以用 anguar.element($0).scope()进行调试

$scope 的生命周期
Creation  --  Watcher registration  --  Model mutation  --
Mutation observation  -- Scope destruction
