Angular 指令

什么是 AngularJS 指令 <出处 菜鸟教程> 更详细 
AngularJS 指令是扩展的 HTML 属性
通过内置的指令来为应用添加功能、带有前缀、ng-

部分常见指令
ng-app 指令初始化一个 AngularJS 应用程序。
ng-controller 指令用于为你的应用添加控制器。
在控制器中，你可以编写代码，制作函数和变量，并使用 scope 对象来访问。
ng-model 指令把元素值（比如输入域的值）绑定到应用程序。

★AngularJS 允许你自定义指令。
ng-cloak 和 ng-bind 指令处理在页面未成功加载AngularJS 未能解释表达式{{}} 
页面在刚开始加载会出现angular 的{{}} 、这会影响页面展示效果(不美观)
去掉表达式{{}} 可以使用 ng-cloak 和 ng-bind

1.元素中添加ng-bind可以省略{{}}、同样的效果
<p ng-bind="msg"></p>

2.元素中添加ng-cloak 并class="ng-cloak"
<div ng-cloak class="ng-chlock">{{aaa}}</div>
原理是刚加载的时候、class类ng-cloak隐藏{{}}两个大括号、然后再删除class

★它们的区别、什么时候该使用 ng-cloak 或 ng-bind
从AngularJS库加载上考虑、一般在首页Index引入AngularJS库、
ng-bind就在首页上使用、若加载不成功也只会不显示AngularJS的内容、
而不会出现{{}}

ng-csp 修改内容的安全策略 
★思考? 这的使用场景

ng-csp 指令用于修改 AngularJS 的安全策略。
如果使用了 ng-csp 指令， AngularJS 将不会执行eval 函数，
  这样就无法注入内联样式。
设置 ng-csp 指令为 no-unsafe-eval, 将阻止 AngularJS 执行 eval 函数，
  但允许注入内联样式。
设置 ng-csp 指令为 no-inline-style, 将阻止 AngularJS 注入内联样式，
  但允许 执行 eval 函数。
如果开发 Google Chrome 扩展或 Windows 应用 ng-csp 指令是必须的。
注意:ng-csp 指令不会影响 JavaScript,但会修改 AngularJS 的工作方式，
这就意味着: 你仍然可以编写 eval 函数, 且也可以正常执行, 但是 AngularJS 
不能执行它自己的 eval 函数。如果采用兼容模式，会降低 30% 的性能。

扩展
当开发如google浏览器的扩展时候这个就必须使用。

CSP禁止应用程序使用eval和Function(string)生成的函数。如果我们需要兼容，
我们只需要使用$parse执行getterfn而不违反这些限制。

AngularJs使用Function(string)以最佳的速度生成功能。使用ngScp指令会导致
